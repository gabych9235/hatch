# Code of Conduct

## Our Promise

Hatch is built for everyone — WordPress users, frontend devs, agency teams, indie writers, non-technical site owners. To keep this community welcoming, we ask everyone to follow this code.

## Our Standards

**Examples of welcome behavior:**
- Be patient with people learning headless WordPress for the first time
- Critique code, designs, and ideas — never people
- Accept constructive feedback gracefully
- Help newcomers find the docs / FAQ / discussions
- Default to assuming good intent
- English not required to be perfect — broken English is fine, ideas matter more

**Examples of unwelcome behavior:**
- Personal attacks, insults, harassment
- Trolling, deliberately inflammatory comments
- Sexual language or imagery
- Doxxing (publishing private information)
- Discrimination based on race, religion, gender identity, sexual orientation, nationality, ability, age, body type, or appearance
- "RTFM" responses without pointing to the relevant FM
- Sustained disruption of discussions

## Where this applies

- GitHub repository (issues, PRs, discussions, code reviews)
- Any official Hatch channel (Discord/Slack if we add one)
- Hatch-related events / talks / livestreams

## Enforcement

If you experience or witness behavior that violates this code, email **support@adityaarsharma.com**. Reports stay confidential.

Maintainer responses (escalating):
1. Private warning + request to edit/delete the offending content
2. Temporary ban from interaction (issues / discussions / PRs) — 7 days
3. Permanent ban from the project

We reserve the right to remove comments, commits, code, and contributions that don't align with this code.

## Maintainer commitment

Maintainers will:
- Lead by example
- Respond to reports promptly
- Apply this code consistently regardless of contributor status
- Listen to community feedback on the code itself
